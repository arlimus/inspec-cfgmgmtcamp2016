Example compliance requirement

SSH supports two different protocol versions. The original version, SSHv1, was subject to a number of security issues. Please use SSHv2 instead to avoid these.

Express this test in code

            
describe sshd_config do
  its('Protocol') { should cmp 2 }
end

Express the compliance control in code

            
control 'ssh-1234' do
  impact 1.0
  title 'Server: Set protocol version to SSHv2'
  desc "
    Set the SSH protocol version to 2. Don't use legacy
    insecure SSHv1 connections anymore...
  "

  describe sshd_config do
    its('Protocol') { should eq('2') }
  end
end

Implementation example (Chef)

            
template '/etc/ssh/sshd_config' do
  source 'opensshd.conf.erb'
  mode '0600'
  owner 'root'
  group 'root'
  variables(
    protocol: '2',
    ...

Example no.2: Do not use old SSL

            
describe ssl(port: 443).where { cipher =~ /SSL/ } do
  it { should_not be_enabled }
end

Full compliance control

            
control 'ssl-test-01' do
  title 'Disable SSL on port 443'
  desc  'You should not run old versions of SSL anymore...'
  ref   'Internal Compliance Rules, v1.0, 5.5.1'
  impact 1.0

  describe ssl(port: 443).where { cipher =~ /SSL/ } do
    it { should_not be_enabled }
  end
end

Continuous compliance

  • Test continuously: With CI/CD
    to fail in your pipeline, not production
  • Monitor production

Successful compliance automation

  • Pick a tool that is understood by everyone involved
  • Provide access to fail fast, fail small
  • Open opportunities to collaborate
  • Automate everything you can
